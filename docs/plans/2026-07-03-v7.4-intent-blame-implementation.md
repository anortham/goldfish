# Goldfish 7.4.0 — Intent-Blame Recall + Brief Refresh Nudge Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use razorback:subagent-driven-development when subagent delegation is available. Fall back to razorback:executing-plans for single-task, tightly-sequential, or no-delegation runs.

**Goal:** Ship 7.4.0 with a brief refresh nudge (catches zombie briefs) and `file:`/`symbol:` recall filters (intent-blame queries over already-captured frontmatter).

**Architecture:** Both features extend recall's existing filter/present pipeline following the 7.2.0 `type`/`tags` and 7.1.0 staleness patterns. No new tools, no storage change, non-destructive. Design spec: `docs/plans/2026-07-03-v7.4-intent-blame-design.md` (approved, Codex-reviewed).

**Tech Stack:** Bun, TypeScript, MCP SDK, Orama (untouched), bun test.

**Architecture Quality:** Approved shape: two optional `recall` params (`file`, `symbol`) + one optional `RecallResult` field (`briefRefresh`); changes stay inside `src/recall.ts` filter/present pipeline, `src/types.ts`, `src/tools.ts` schema, `src/handlers/recall.ts` display. Rejected: new `blame` tool, success-criteria parsing, auto-archiving. Architecture risk: low.

## Global Constraints

- TDD mandatory: every task writes failing tests first, commits test + implementation together.
- UTC timestamps everywhere; atomic write-then-rename for any file writes (no new writes expected).
- 3-tool constraint: no new MCP tools; no new `brief` actions.
- Claude Code 2,000-char cap on server instructions and each tool description (enforced by tests/server.test.ts) — the recall description gains two params and MUST stay ≤ 2,000 chars; trim examples if needed.
- Thresholds: `STALE_BRIEF_DAYS = 7` (existing, unchanged value), `BRIEF_REFRESH_DAYS = 14` (new).
- Refresh nudge display line (exact copy): `⚠️ Active brief not updated in {N}d — still the direction? Update it to reaffirm, or complete/archive it.`
- Non-destructive: recall never mutates brief status on disk.
- Version 7.4.0 on all five surfaces: `src/server.ts` `SERVER_VERSION`, `package.json`, `.claude-plugin/plugin.json`, `.claude-plugin/marketplace.json` (`plugins[0].version`), README banner. Plus a `## [7.4.0]` CHANGELOG section.
- `skills/` is canonical; `.agents/skills/` is regenerated via `bun run scripts/sync-agent-skills.ts` — never hand-edit the mirror.
- No em dashes in script headers (project rule); keep docs free of hardcoded test counts.

### Task 1: Abandoned-staleness activity includes `brief.updated`

**Files:**
- Modify: `src/recall.ts:36-66` (`resolveActiveBrief`)
- Test: `tests/recall.test.ts`

**Interfaces:**
- Consumes: `Brief.updated` (existing field, refreshed by `updateBrief` in `src/briefs.ts`), `findLatestCheckpointTimestampForBrief` (existing).
- Produces: abandoned-staleness activity = `max(brief.created, brief.updated, latest referencing checkpoint)`. Task 2 builds on this exact rule.

**What to build:** Fix the Codex-review coherence gap — updating an abandoned brief must un-suppress it. Fold `brief.updated` into the `lastActivity` computation before the `STALE_BRIEF_DAYS` comparison.

**Approach:** Compare ISO strings via `new Date(...).getTime()`; take the max of the three timestamps (updated may be absent on legacy briefs — guard with `brief.updated ?? brief.created`). Keep the bounded date-dir scan unchanged.

**Acceptance criteria:**
- [x] Regression test: brief created 30d ago, updated 2d ago, no checkpoints in 10d → NOT suppressed, no `staleBrief` notice.
- [x] Existing behavior preserved: brief with no update and no checkpoints for >7d → suppressed with `staleBrief` notice.
- [x] `bun test recall` passes, committed.

### Task 2: Brief refresh nudge (zombie briefs)

**Files:**
- Modify: `src/types.ts` (add `BriefRefreshNotice`; add `briefRefresh?: BriefRefreshNotice | null` to `RecallResult`)
- Modify: `src/recall.ts` (`resolveActiveBrief` return shape + `BRIEF_REFRESH_DAYS = 14` constant; thread through `recallFromWorkspace` and single-workspace `recall()` return)
- Modify: `src/handlers/recall.ts` (append nudge line under the active-brief section)
- Test: `tests/recall.test.ts`, `tests/handlers.test.ts`

**Interfaces:**
- Consumes: Task 1's activity rule (a zombie brief is by definition NOT abandoned-stale).
- Produces: `BriefRefreshNotice { id: string; title: string; updated: string; daysSinceUpdated: number }`; `RecallResult.briefRefresh`. Mutual exclusivity contract: `staleBrief` set ⇒ `activeBrief` null and `briefRefresh` null; `briefRefresh` set ⇒ `activeBrief` non-null.

**What to build:** When the active brief passes the abandoned check but `brief.updated` is older than 14 days, return the brief plus a `briefRefresh` notice. Handler appends the exact nudge line from Global Constraints under the brief body. Applies to single-workspace recall (including `limit: 0` brief-only mode); cross-workspace recall unchanged.

**Approach:** Extend `resolveActiveBrief`'s return to `{ activeBrief, staleBrief, briefRefresh }`. Handler: after `formatActiveBrief`, if `result.briefRefresh`, push blank line + nudge. Header suffix stays `+ active brief`.

**Acceptance criteria:**
- [x] Brief updated 20d ago with a checkpoint 1d ago → body shown + `briefRefresh` notice with `daysSinceUpdated` = 20.
- [x] Brief updated 5d ago → no notice.
- [x] Abandoned brief (>7d no activity) → `staleBrief` only; `briefRefresh` never double-fires.
- [x] `limit: 0` recall carries the notice; handler output contains the exact nudge line.
- [x] `bun test recall handlers` passes, committed.

### Task 3: `file:` recall filter with shared path normalization

**Files:**
- Modify: `src/types.ts` (`RecallOptions.file`, `RecallInput`, `RecallArgs`)
- Modify: `src/recall.ts` (normalization helper, `normalizeRecallOptions`, `hasStructuredFilter`, `applyStructuredFilters`)
- Modify: `src/tools.ts` (recall schema: `file` property + one description example)
- Test: `tests/recall.test.ts`, `tests/server.test.ts` (2k cap still green)

**Interfaces:**
- Consumes: `Checkpoint.git.files` (existing frontmatter).
- Produces: `recall({ file: string })` — case-insensitive path-suffix match with `/`-boundary guard. Exports (module-internal) `normalizeFilePathForMatch(path: string): string` used on BOTH query and stored paths: trim, `\` → `/`, strip leading `./` and leading/trailing `/`, lowercase. Match: `stored === query || stored.endsWith('/' + query)`. Task 4 reuses the structured-filter wiring; Task 5 reads `options.file`.

**What to build:** Full vertical slice: type, normalization, filter, full-corpus scan trigger, tool schema. Checkpoints without `git.files` never match.

**Approach:** Add `Boolean(options.file)` to `hasStructuredFilter`; add the file predicate to `applyStructuredFilters`. Normalize the query once in `normalizeRecallOptions`. Works in single- and cross-workspace recall automatically (both call `applyStructuredFilters` via `loadWorkspaceCheckpoints`). If the recall description nears 2,000 chars, trim an existing example rather than omitting the new param.

**Acceptance criteria:**
- [x] `file: "recall.ts"` matches checkpoints with `src/recall.ts` AND `src/handlers/recall.ts`; `file: "handlers/recall.ts"` matches only the latter; `file: "all.ts"` matches neither (boundary guard).
- [x] `file: "src\\recall.ts"` (Windows-style query) matches stored `src/recall.ts`; case-insensitive both sides.
- [x] Filter scans full corpus (older match beyond most-recent-5 window is found) and composes with `search`, `type`, `tags`, and date windows; works with `workspace: "all"`.
- [x] Recall tool description ≤ 2,000 chars (tests/server.test.ts green).
- [x] `bun test recall server` passes, committed.
