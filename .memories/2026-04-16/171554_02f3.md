## Added request-time roots-aware workspace resolution

Implemented the first portability task in the feature worktree.

- **WHAT:** Extended workspace resolution to understand MCP roots, added file-root parsing helpers, introduced `createServer()` with request-time workspace hydration, and covered the behavior with in-memory MCP tests.
- **WHY:** Codex, OpenCode, and VS Code can provide roots dynamically, and Goldfish needed to bind tool calls to the correct workspace without forcing `GOLDFISH_WORKSPACE` in every client.
- **HOW:** Added pure helpers in `src/workspace.ts` for `explicit > env > roots > cwd`, cached roots per session in `src/server.ts`, cleared that cache on `notifications/roots/list_changed`, and drove the server behavior through real MCP client/server tests using the SDK's in-memory transport.
- **IMPACT:** Goldfish now picks the right workspace in roots-aware clients while keeping Claude and cwd fallback behavior intact. The new tests guard lazy lookup, cache reuse, invalidation, and fallback behavior.
