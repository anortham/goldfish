## Verified harness docs and corrected the VS Code section

Checked the current README against official docs for Codex, OpenCode, VS Code, and Claude-related plugin compatibility.

- **WHAT:** Verified the Codex `config.toml` MCP setup, OpenCode `opencode.json` local MCP setup, OpenCode `.agents/skills` discovery, and VS Code `.vscode/mcp.json` setup against official docs. Found one meaningful gap: VS Code now supports Claude-format agent plugins, which means Goldfish can preserve skills and hooks there through the plugin path instead of only through MCP.
- **WHY:** The portability docs were meant to make Goldfish easier to use across harnesses. If the README undersells a supported install path, users miss functionality we already ship.
- **HOW:** Compared the README with official docs, then updated the VS Code section to mention agent plugins preview, Claude-format plugin compatibility, and the `chat.pluginLocations` / `chat.plugins.marketplaces` paths. Added a regression assertion so the README keeps mentioning the plugin path.
- **IMPACT:** The README now matches the current harness docs more closely, especially for VS Code users who want skills and hooks instead of MCP-only setup.
