## Added repo-local skill mirroring for Codex and OpenCode

Finished the second portability slice in the feature worktree.

- **WHAT:** Added `scripts/sync-agent-skills.ts`, introduced a `sync:agent-skills` package script, and generated `.agents/skills/*/SKILL.md` mirrors from the canonical `skills/` tree.
- **WHY:** Codex and OpenCode discover repo-local skills from `.agents/skills`, while Goldfish still needs `skills/` as the single authored source for the Claude plugin.
- **HOW:** Wrote metadata regression tests first, then implemented a sync script that copies canonical `SKILL.md` files into `.agents/skills` and prunes stale mirrored directories.
- **IMPACT:** Repo users on Codex and OpenCode can discover Goldfish skills without extra install steps, and the tests now catch mirror drift before release.
