## Task 1 complete: brief-first storage compatibility

### WHAT
Implemented the first migration slice for the `plan` -> `brief` rename.

- Added `getBriefsDir()` in `src/workspace.ts`
- Updated `ensureMemoriesDir()` to create both `.memories/briefs/` and `.memories/plans/`
- Added brief-facing type aliases and compatibility fields in `src/types.ts`
- Reworked `src/plans.ts` so new writes go to brief storage while reads still support legacy plan files and `.active-plan`
- Added compatibility exports such as `saveBrief`, `getBrief`, and `getActiveBrief`
- Expanded tests to cover brief storage, legacy fallback reads, and `.active-brief`

### WHY
Later tasks depend on a storage layer that understands both the new brief model and old plan data. Without this, the public rename would break existing files and callers.

### HOW
- Wrote failing tests first for `getBriefsDir`, brief directory creation, brief storage writes, legacy plan reads, and active brief fallback behavior
- Implemented the minimal storage helpers and compatibility wrappers to make those tests pass
- Kept the legacy `plan` API intact while redirecting fresh writes into brief storage

### IMPACT
- Goldfish can now write new forward-looking artifacts into `.memories/briefs/`
- Existing `.memories/plans/` files and `.active-plan` markers still work
- Task 2 can build the `brief` MCP surface on top of a stable compatibility layer
