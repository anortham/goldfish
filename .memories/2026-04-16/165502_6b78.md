## Wrote cross-client portability design

Defined the next product-level direction for Goldfish after the `brief` migration.

- **What:** Added a design doc for cross-client positioning, MCP roots-based workspace detection, thin client adapters for Claude/Codex/OpenCode/VS Code, and a narrower `standup` scope built only from briefs and checkpoints.
- **Why:** Goldfish already works as an MCP server outside Claude Code, but the repo still markets and packages it in a Claude-first way. At the same time, pulling harness plans into standup would undercut the move away from Goldfish-owned planning.
- **How:** Wrote `docs/plans/2026-04-16-cross-client-portability-design.md` with goals, non-goals, compatibility constraints, request-time roots resolution rules, and rejected alternatives.
- **Impact:** Gives implementation work a clear target without breaking existing Claude plugin users and keeps standup aligned with Goldfish's memory-only role.
