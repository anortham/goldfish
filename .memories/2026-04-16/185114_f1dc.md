## v7.0.0 strategic direction set; first design doc written

Long brainstorming session with the user about goldfish positioning in late 2026 produced a consolidated v7.0.0 sprint plan. Both Gemini and Codex (dispatched as background second opinions) independently converged with the user's intuition and my own read.

### Strategic repositioning
Goldfish is no longer "memory" (1M context wins that frame). It's the **evidence ledger / git for intent**: source-controlled, harness-agnostic, durable record of why things changed. Three legs: checkpoints (evidence), briefs (current strategic thesis), cross-project recall (multi-project ergonomic).

### v7.0.0 subtract sprint (agreed scope)
- Replace fuse.js + semantic stack with Orama BM25 (this design doc covers it)
- Delete SessionStart + PreCompact hooks (Claude-Code-only, intrusive on cheap context)
- Delete consolidation routine (token math is net-negative as wired)
- Retire `plan` tool clean (no compat alias), rename `plans.ts` → `briefs.ts`
- Cleanup: `docs/JULIE_BUNDLING_STRATEGY.md`, `docs/JULIE_INTEGRATION_PLAN.md`, `docs/RAG_PLANNING.md`, `docs/superpowers/`, old revival plan docs, `test-julie-integration.ts`
- Build: handoff layer (`/handoff` skill or `recall({ handoff: true })`) producing returning-engineer summary from active brief + recent checkpoints + git delta

### First design doc
`docs/plans/2026-04-16-orama-search-replacement-design.md` covers the Orama swap (one piece of the subtract sprint). Empirical comparison on 94 real checkpoints showed fuse silently returned zero results on natural-language queries while Orama BM25 found relevant hits on every query. Catch from spec self-review: `getSemanticWorkspaceKey` is a misleadingly named generic hash function used by `getConsolidationStatePath`; renamed to `getWorkspaceHashKey` instead of deleted.

### Key tradeoffs decided
- Subtract first (path 1), build handoff in 7.1
- No backwards compatibility for plan tool retirement
- Semantic embeddings deferred (could come back as opt-in API-based via Orama's vector layer)
- Hooks are not the product surface for cross-client UX
