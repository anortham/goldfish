## Added canonical brief MCP surface

- **WHAT:** Added a new `brief` MCP tool and handler as the canonical forward-looking surface, while keeping `plan` as a compatibility alias. Updated tool descriptions, server dispatch, and exported handlers so both names work during the migration window.
- **WHY:** Goldfish is moving away from mirrored harness planning toward a compact strategic brief. The public MCP surface needed to reflect that shift without breaking older callers in the same release.
- **HOW:** Created `src/handlers/brief.ts` with shared logic, routed `handlePlan` through the same path, registered `brief` in the tool list and server dispatch, and rewrote instructions/tool descriptions to present `plan` as a compatibility alias instead of the primary interface.
- **IMPACT:** Agents can now discover and use `brief` directly, older `plan` calls still resolve, and the server surface no longer tells harnesses to copy every approved plan into Goldfish.
