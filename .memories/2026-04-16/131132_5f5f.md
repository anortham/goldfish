## Reset hooks and skill layer around brief-first semantics

- **WHAT:** Removed the `ExitPlanMode` mirroring hook, added canonical `/brief` and `/brief-status` skills, and rewrote the existing plan-oriented skills into compatibility aliases or brief-first instructions.
- **WHY:** Goldfish cannot keep claiming that harnesses own execution planning while the hook layer still copies every approved harness plan and the skills still teach agents to use Goldfish as a second planner.
- **HOW:** Deleted the `PostToolUse` hook for `ExitPlanMode`, updated the session-start reminder to talk about active briefs, created new brief skills, and replaced the old plan-heavy skill bodies with concise guidance that treats briefs as durable direction, `docs/plans/` as execution detail, and checkpoints as evidence.
- **IMPACT:** New sessions no longer get told to mirror harness plans into Goldfish, the primary slash-command vocabulary is now brief-first, and older `/plan` workflows still have a compatibility path without reinforcing the old model.
