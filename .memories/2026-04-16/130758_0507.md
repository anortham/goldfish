## Renamed downstream recall and checkpoint surfaces to brief language

- **WHAT:** Switched checkpoint affinity, recall results, consolidation payloads, digests, ranking metadata, and handler wording over to `brief` terminology while preserving legacy `plan` compatibility.
- **WHY:** Task 2 exposed the new `brief` tool, but the rest of the memory flow still wrote `planId`, returned `activePlan`, and rendered `Active Plan` text. That split would have left the rename half-finished and confusing.
- **HOW:** Added `briefId` to checkpoint data, made new checkpoint frontmatter write `briefId`, mapped legacy `planId` frontmatter back into the same in-memory affinity, normalized recall filters so `briefId` and `planId` resolve to one value, exposed `activeBrief` alongside the compatibility alias, updated consolidation to point at active brief files, and taught digests/ranking to include the brief identifier.
- **IMPACT:** New checkpoints and recall output now use brief language end to end, older checkpoint files still filter correctly, and consolidation prompts follow the new brief-first model without losing one-release compatibility for older callers.
