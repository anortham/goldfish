## Implemented backward-compatible structured memory schema upgrade

### WHAT
Added a richer checkpoint schema to improve decision-level recall quality, while keeping all existing memories fully valid.

### WHY
Current memory retrieval is strong for session continuation but weaker for "why did we do this?" queries. Structured metadata (decision/context/impact/etc.) improves searchability and interpretability for historical reasoning.

### HOW
- Extended checkpoint types with optional fields:
  - `type` (`checkpoint|decision|incident|learning`)
  - `context`, `decision`, `alternatives`, `impact`, `evidence`, `symbols`, `next`, `confidence`, `unknowns`
- Updated serialization/parsing in `src/checkpoints.ts`:
  - Writes new fields to YAML frontmatter only when present
  - Parses new fields safely from frontmatter
  - Normalizes arrays and confidence (1-5)
- Updated save flow and handler validation:
  - `handleCheckpoint` forwards structured args
  - Rejects invalid confidence values outside 1-5
- Improved recall relevance in `src/recall.ts`:
  - Added structured fields to Fuse search keys with weighting
- Updated output and tool schema:
  - `src/handlers/recall.ts` now renders key structured fields when present
  - `src/tools.ts` `checkpoint` input schema now exposes structured fields
- Added tests across checkpoints/recall/handlers/server schema coverage

### IMPACT
- New memories can be much more informative for codebase/decision understanding
- Existing historical memories require no migration and continue to parse/recall exactly as before
- Structured fields are optional, so adoption can be gradual
- Confidence in change: **91/100** (targeted tests pass)
