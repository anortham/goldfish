## Implemented three-layer enforcement for structured memory schema

Added enforcement for the new structured checkpoint fields (`type`, `decision`, `alternatives`, `context`, `evidence`, `impact`, `symbols`, `next`, `confidence`, `unknowns`) via three complementary approaches:

### 1. Handler Nudges (runtime feedback)
- Checkpoint handler returns `üí°` warnings when high-value fields are missing based on `type`
- Decision checkpoints warn about missing `decision` + `alternatives`
- Incident checkpoints warn about missing `context` + `evidence`
- Learning checkpoints warn about missing `impact`
- No warnings when fields are present or no type is specified

### 2. Server Instructions (`instructions.ts`)
- Added "Structured Fields (USE THEM)" section mapping checkpoint types to recommended fields
- Clear `‚Üê ALWAYS provide` markers for required fields per type
- Explains that structured fields are searchable

### 3. Tool Description (`tools.ts`)
- Added "CLASSIFY YOUR CHECKPOINTS" section to the checkpoint tool description
- Concise type‚Üífield mapping: decision‚Üídecision+alternatives, incident‚Üícontext+evidence, learning‚Üíimpact
- Emphasizes searchability benefit

- 5 new handler tests (3 warning cases, 2 no-warning cases)
