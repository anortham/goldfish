## Completed Tasks 3 & 4: Replace all process.cwd() with resolveWorkspace()

### WHAT
- **Task 3** (commit `90e41f5`): Replaced `process.cwd()` fallbacks in all 3 handler files
  - `src/handlers/checkpoint.ts`: `workspace || process.cwd()` -> `resolveWorkspace(workspace)`
  - `src/handlers/plan.ts`: `wsArg || process.cwd()` -> `resolveWorkspace(wsArg)`
  - `src/handlers/recall.ts`: Replaced local `resolveWorkspacePath()` function to delegate to `resolveWorkspace()`

- **Task 4** (commit `6ad934a`): Replaced `process.cwd()` fallbacks in all 4 core modules
  - `src/checkpoints.ts`: `input.workspace || process.cwd()` -> `resolveWorkspace(input.workspace)`
  - `src/plans.ts`: `input.workspace || process.cwd()` -> `resolveWorkspace(input.workspace)`
  - `src/recall.ts`: `workspace === 'current' ? process.cwd() : workspace` -> `resolveWorkspace()`
  - `src/workspace.ts`: `getMemoriesDir()` now uses `resolveWorkspace()` instead of bare `process.cwd()`
  - Updated JSDoc comments in workspace.ts to reflect new fallback chain

### WHY
Centralizes all workspace resolution into `resolveWorkspace()` which provides the priority chain:
`explicit arg > GOLDFISH_WORKSPACE env > process.cwd()`

This enables VS Code / Copilot MCP integration where `cwd` isn't the project root.

### HOW
- Direct search-and-replace of each `process.cwd()` site with `resolveWorkspace()` call
- Added imports where needed
- 265 tests pass after each commit

### IMPACT
- `process.cwd()` is now called in exactly ONE place in `src/`: inside `resolveWorkspace()` on line 59 of `workspace.ts`
- All 7 previously scattered fallback sites now go through the centralized resolver
- Tasks 1-4 of the v5.1 plan are now complete
