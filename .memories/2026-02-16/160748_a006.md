## v5.0.3 committed and pushed

### What
Legacy Julie format support, CRLF handling, recall diagnostics.

### Key changes
- **`parseJsonCheckpoint()`**: New function for old Julie `.json` checkpoint files
- **`normalizeTimestamp()`**: Unix seconds/ms → ISO 8601
- **`normalizeGit()`**: `files_changed`/`dirty`/`type` normalization
- **CRLF**: Both `parseCheckpointFile()` and `parsePlanFile()` now normalize `\r\n` → `\n`
- **Recall diagnostics**: Response includes `resolvedPath`, `memoriesDir`, `memoriesExists`
- **242 tests**, all passing

### Impact
Goldfish can now read memories created by the old Julie checkpoint system, unblocking the recall-not-finding-anything bug on the Julie project.
