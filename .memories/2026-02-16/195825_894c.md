## Updated /plan-status skill with dual-source plan awareness (Task 8)

- **WHAT:** Replaced `skills/plan-status/SKILL.md` with new content that checks both `.memories/plans/` (Goldfish plans) and `docs/plans/*.md` (project plan docs)
- **WHY:** The user uses a superpowers plugin that writes implementation plans to `docs/plans/`. The plan-status skill previously only knew about Goldfish plans, giving incomplete assessments.
- **HOW:** Updated Step 1 from single-source ("Get the active plan") to dual-source ("Gather plans from ALL sources"). Added Source 1/Source 2 structure, docs/plans status verification against checkpoints, source-attributed report headers, a "Multiple Plans" section for handling both simultaneously, and two new critical rules about checking both sources and attributing clearly.
- **IMPACT:** Plan status assessments now cover all plan sources, with honest verification of docs/plans status headers against actual checkpoint evidence
