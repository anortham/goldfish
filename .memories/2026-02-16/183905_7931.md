## All v5.0.7 nits resolved — 258 tests passing

Fixed all 6 remaining nit-level issues from the code review:

### Fixes
- **Plan frontmatter regex**: Accepts single or double newline after `---` (was double-only). Added frontmatter field validation to catch garbage YAML that the parser silently accepts.
- **`deletePlan` TOCTOU**: Moved `.active-plan` read + clear inside the lock, eliminating the race where another process could activate a different plan between read and delete.
- **`activePlan` type mismatch**: Changed `RecallResult.activePlan` from `Plan | undefined` to `Plan | null | undefined` to match what `getActivePlan()` actually returns.
- **`normalizeTimestamp` garbage input**: Now falls back to `new Date().toISOString()` for null/undefined/empty instead of returning `"null"` string.
- **Lock timeout test**: Replaced no-op placeholder with real test verifying error message format.
- **BOM handling**: Both `parseCheckpointFile` and `parsePlanFile` now strip `\uFEFF` BOM prefix (Windows Notepad edge case).

### New tests added (5)
- Plan single-newline separator parsing
- Plan BOM stripping
- Checkpoint BOM stripping
- Checkpoint null timestamp handling
- `deletePlan` doesn't clear unrelated active plan
