## Brainstorming session — feature audit complete

Thorough review of Goldfish's current feature set with the user to determine what stays, what changes, and what's out of scope.

### Key Decisions
- **Keep plan tool** — portability for non-Claude Code clients (Codex, VS Code/Copilot)
- **No task management** — Claude Code handles it natively, would conflict
- **No DevOps integration** — work items are too high-level ("work on Lab Handbook enhancements")
- **Anamnesis is separate** — agent identity/wisdom lives in ~/Source/claude, not Goldfish

### Changes Identified
1. **Add `/plan` skill** — guide agents through plan tool usage
2. **Update `/standup` skill** — also scan `docs/plans/` for superpowers-written plans, infer completion status from checkpoints
3. **Update `/plan-status` skill** — same dual-source plan awareness
4. **Add `GOLDFISH_WORKSPACE` env var** — VS Code workspace resolution (already specced in TODO.md)

### Context
- User bounces between multiple projects daily, forgetful, needs standup reports
- Original Goldfish standup was heavily used and valued
- Superpowers plugin writes plans to `docs/plans/` with Status headers
- Claude Code auto-memory has zero overlap with Goldfish (stable knowledge vs work tracking)
