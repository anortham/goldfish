## Replaced JSON tool output with readable markdown

All three MCP tool handlers now return readable markdown instead of `JSON.stringify()`.

### Changes
- **`src/handlers/checkpoint.ts`** — One-line-per-field format: fish emoji header, time, branch, tags, files (capped at 10)
- **`src/handlers/recall.ts`** — Fish emoji header with count + diagnostics, active plan as H2 section, checkpoints as H3 entries with timestamp/id/tags/description
- **`src/handlers/plan.ts`** — One-liner confirmations for save/activate/update/complete; full markdown for get; bullet list for list
- **`tests/handlers.test.ts`** — Rewrote 25 tests: removed all JSON.parse(), uses .toContain()/.toMatch()/.not.toStartWith('{')
- **`tests/server.test.ts`** — Rewrote all JSON.parse() assertions to match markdown output

### Why
JSON wastes tokens on quotes, brackets, colons, and field names. The agent reads tool output as text — never parses it. Markdown is more compact and readable.

### Result
- 246 tests passing (all 10 test files)
- Significant token savings on every recall/checkpoint/plan call
