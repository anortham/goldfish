## Standup skill timeframe args + week unit in parseSince

**WHAT:** `/standup 7d`, `/standup 1w`, `/standup 3d` now work. Added `w` (week) unit to `parseSince()` in `src/recall.ts` and taught the standup skill to pass user-supplied timeframes through to recall.

**WHY:** User wanted Monday catch-up standups covering the previous week (or a weekend), but the standup skill hardcoded `recall({ workspace: "all", days: 1 })` and `since` only accepted `m`/`h`/`d` units.

**HOW:**
- TDD: failing tests for `parseSince('1w')` / `parseSince('2w')` in `tests/recall.test.ts`, then extended the unit regex to `[mhdw]` with a `7 * 24h` multiplier. Backwards-compatible, additive.
- New "Timeframe" section in `skills/standup/SKILL.md` (mirrored to `.agents/skills/standup/SKILL.md`): default `days: 1`, pass user args as `since`/`days`, weekend → `3d`, weekly → `1w`.
- Updated `since` format docs in `src/tools.ts` recall description, CLAUDE.md, AGENTS.md, CONTRIBUTING.md, and `docs/goldfish-checkpoint.instructions-vs-code.md`.

**IMPACT:** Skills accept free-text args, so no plumbing was needed — only skill guidance plus real `1w` support in the parser. Full suite 619 pass, tsc clean; 2k tool-description cap test still passes.
