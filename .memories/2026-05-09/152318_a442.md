## Reject filesystem-root cwd fallback in workspace hydration

Desktop MCP clients (VS Code, Codex Desktop) launch goldfish with `cwd=/`. Without `GOLDFISH_WORKSPACE` and with failing/empty `roots/list`, goldfish was silently falling through to `process.cwd()` and trying to create `/.memories/` at the filesystem root. macOS's protected `/` masked the bug with EROFS; Linux or non-protected mounts would have actually polluted the filesystem root.

- **Origin:** Adapted from Julie's uncommitted fix on `~/source/julie` (`reject_cwd_filesystem_root_startup_hint` in `src/handler.rs`)
- **Scope simplified vs Julie:** No deferred-indexing logic (goldfish reads markdown lazily per call), no `client_supports_workspace_roots` capability tracking (goldfish already does lazy per-call `roots/list` with try/catch)
- **API addition:** `resolveWorkspaceWithSource()` in `src/workspace.ts` returns `{ path, source: 'explicit'|'env'|'roots'|'cwd' }`. Existing `resolveWorkspace()` delegates to it (no signature change)
- **Guard location:** `hydrateWorkspaceArguments` in `src/server.ts` — rejects only when `source==='cwd' AND dirname(path)===path`. `dirname(p)===p` is the cross-platform "no parent" check (handles `/` on Unix and `C:\` on Windows). Explicit `GOLDFISH_WORKSPACE=/` is allowed (user's call)
- **Error message:** "Refusing to use filesystem root (/) as workspace from process cwd. Set GOLDFISH_WORKSPACE to your project path or open a project folder in your MCP client."
- **Tests:** 6 new in `tests/workspace.test.ts` (source-tagging per branch), 2 new in `tests/server.test.ts` (integration: rejection + explicit-env-override-allowed). 455/455 pass, typecheck clean
- **Codex consensus:** Attempted twice via `/codex-cli`, both hung (likely auth/rate-limit). Self-reviewed instead — see conversation
- **Not yet:** No version bump, no CHANGELOG entry, no commit. Working tree only.

## Impact

Closes the longstanding "goldfish only works with CLI clients" gap. VS Code Desktop and Codex Desktop should now get a clear error pointing to `GOLDFISH_WORKSPACE` instead of silent `/.memories/` writes. Workspace-local config workaround should still work but is no longer mandatory for desktop clients that fail to advertise roots.
