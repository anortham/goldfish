## Hardened cwd workspace fallback and bumped 7.0.4

Goldfish now protects user-level MCP installs from weak `cwd` defaults that point at broad system or home directories.

- **WHAT:** Added source-aware workspace resolution, unsafe cwd fallback detection, server rejection paths, regression coverage, and 7.0.4 release metadata.
- **WHY:** Desktop/user-level MCP launches can lack roots or a proper project cwd, which could otherwise write `.memories/` into `/`, home directories, Windows user roots, or Windows system paths.
- **HOW:** `resolveWorkspaceWithSource()` tags explicit/env/roots/cwd sources; `getUnsafeCwdWorkspaceReason()` rejects only cwd-derived unsafe paths; server hydration keeps explicit `workspace`, `GOLDFISH_WORKSPACE`, and MCP roots as overrides.
- **IMPACT:** Safer user-level install behavior without removing project roots support or explicit overrides.
- **Verified:** `bun test ./tests/workspace.test.ts ./tests/server.test.ts`, `bun run typecheck`, and `bun test` all pass.
