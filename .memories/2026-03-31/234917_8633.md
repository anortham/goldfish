**Chunk 2 (GPT): Bounded search maintenance**
- `SEARCH_MAINTENANCE_MAX_ITEMS=5`, `SEARCH_MAINTENANCE_MAX_MS=100` constants

**Chunk 3 (GPT): Derived cache normalization and recovery**
- `SemanticCacheCorruptionError` with strict schema validation

- Corrupted cache files warn and reset to empty state

**Chunk 4: Lock-safe pruning**
- Added `tryAcquireLock(filePath, timeoutMs)` to lock.ts (returns null instead of throwing)
- `pruneOrphanedSemanticCaches()` now acquires lock before deletion, skips locked caches
- TOCTOU protection: manifest re-read inside the lock

**Chunk 5: Embedder abort and queue hardening**
- Cold-start aborts reject promptly via `withAbortSignal(ensureEmbedder(), signal)`
- Removed eager queue release on abort; queue stays serialized via `finally` only
- Simplified `enqueueEmbedding` API

**Chunk 6: Docs update**
- Updated IMPLEMENTATION.md to describe opportunistic ranking, bounded maintenance, and graceful fallback

### Impact
- Recall never hangs on slow/broken semantic runtime
- Search latency no longer scales with indexing backlog
- Corrupted cache self-heals instead of breaking recall
- Prune can't race active cache operations
- Embedder queue stays serialized under abort
