## Fixed bun install failure caused by prepare script redirects

`bun install` failed with `expected a command or assignment but got: "Redirect"` on the `prepare` lifecycle script.

- **Root cause:** Bun 1.3.0 runs package.json lifecycle scripts through its built-in shell, whose parser rejects two consecutive redirects on one command (`>/dev/null 2>&1`). Isolated by testing each redirect alone (both parse fine) vs. combined (parse error).
- **Fix:** Replaced `>/dev/null 2>&1` with the combined form `&>/dev/null` in the `prepare` script — bun's shell accepts it and it remains valid bash/zsh.
- **Impact:** `bun install` completes clean again and the prepare script correctly sets `core.hooksPath` to `.githooks`.
- **Learning:** Avoid `> file 2>&1` in any package.json script for this project; use `&>` instead since bun's shell executes them on all platforms.
