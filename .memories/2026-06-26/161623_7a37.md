## Released Goldfish 7.3.0

Shipped the registry + parent-walk workspace recovery feature and the four Codex-review fixes, then committed, tagged `v7.3.0`, and pushed to `origin/main`.

- **WHAT:** `feat(workspace): registry + parent-walk recovery, harden unsafe-cwd guard, bump 7.3.0` (commit `f0b4c13`, 14 files, +1910/−26). Tag `v7.3.0` pushed.
- **WHY:** Cursor plugin installs spawn the server with `cwd=home` and never advertise the MCP `roots` capability to plugin-launched servers, so every tool call hard-refused. 7.3.0 recovers a project root before accepting cwd or refusing.
- **HOW:** Codex reviewed the staged diff (background, ~4.4 min) and raised 4 findings; all addressed TDD-style before release: (1, High) a registered `$HOME` could bypass the mutating-tool guard via 4a — now unsafe-checked and dropped for checkpoint/brief; (2) pinned that `chooseRecoveryCandidate` treats symlinked same-dir candidates as equal via realpath; (3) removed machine-specific `.codex/config.toml` from the commit and gitignored `.codex/`; (4) strengthened refusal tests to prove no write occurred + added a `brief` refusal equivalent. Added an end-to-end server test for the registered-home case and verified it fails when the fix is disabled.
- **IMPACT:** 548 tests pass (up from 541), `tsc --noEmit` clean. Release is live. Post-restart verification confirmed the plugin cache is running 7.3.0 and the sharpened refusal message (with "Known projects" list) is active. Noted that in this Cursor session cwd is home and the goldfish project is a *child* of home with ~28 registered projects, so recovery correctly cannot auto-pick — the explicit `workspace:` path works as the documented escape hatch.
