## Implemented P0 fixes from agent-centric review

- **What:** Completed both P0 items:
  - Fixed `plan-status` skill instructions to retrieve the active plan via `recall({ limit: 0 })` instead of invalid `plan({ action: "get" })` without `id`.
  - Synchronized version metadata to `5.1.0` across runtime and package surfaces, and added a regression test to enforce consistency.

- **Why:** These were high-impact trust issues for agent users:
  - The plan-status instruction path could fail at runtime when followed literally.
  - Version drift across docs/plugin/runtime undermined debugging confidence.

- **How:**
  - Updated `skills/plan-status/SKILL.md` to use `mcp__goldfish__recall({ limit: 0 })` for active plan retrieval and added explicit optional `plan.get` example with required `id`.
  - Added `SERVER_VERSION` export in `src/server.ts` and used it in MCP server metadata.
  - Updated `package.json` version to `5.1.0`.
  - Added regression test in `tests/server.test.ts` to assert `SERVER_VERSION` matches both `package.json` and `.claude-plugin/plugin.json` versions.

- **Impact:**
  - Removes a concrete skill/API mismatch that could block agents.
  - Prevents future version skew from silently reappearing.
  - Test suite now has explicit guardrails for release metadata consistency.
