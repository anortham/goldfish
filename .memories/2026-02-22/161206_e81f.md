## Completed agent-centric review of Goldfish

- **What:** Assessed project from the perspective of the coding agent target user, focusing on usability, reliability, and instruction ergonomics.
- **Why:** Agent experience quality determines whether memory tooling is used consistently and safely in real workflows.
- **How:** Reviewed `README.md`, `CONTRIBUTING.md`, `CLAUDE.md`, core server/tool/handler code, hook config, and skills; validated test health with `bun test` (265 passing).
- **Key findings:**
  - Strong architecture and docs for agent onboarding and predictable use.
  - Critical API/skill mismatch: `skills/plan-status/SKILL.md` calls `plan({ action: "get" })` without `id`, but handler requires `id`.
  - Version drift across surfaces (`README`/plugin says 5.1.0 vs server/package 5.0.7).
  - Some behavioral instructions are overly absolute (e.g., "trust recall, do not verify"), which can conflict with safety-first agent behavior.
- **Impact:** Provides concrete, prioritized improvements to reduce agent friction and increase trust/consistency.
