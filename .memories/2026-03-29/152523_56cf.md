## Stabilization pass design approved (v6.5.0)

Completed brainstorming session with three-way review (Claude, Gemini, Codex) of the Goldfish project at v6.4.0.

### Scope

**Tier 1 (bugs):**
- Hook stale-counting uses file mtime without 30-day age filter; disagrees with recall/consolidate
- Registry ignores GOLDFISH_HOME env var; hardcoded to ~/.goldfish
- Stale artifacts: .memories/MEMORY.md, .memories/.last-consolidated, stale active plan

**Tier 2 (quality):**
- recall.ts split: extract ranking logic to src/ranking.ts (~200 lines out of 802)
- Handler args: any typing: add typed arg interfaces per handler in types.ts
- Docs patches: README, IMPLEMENTATION.md, CONTRIBUTING.md all describe 5.x
- TODO.md cleanup: strip completed items and historical fix logs, keep only live backlog

### Key decisions
- Approach B (full stabilization, sequential commits) over A (minimal) or C (parallel worktrees)
- Option 1 (extract ranking only) over Option 2 (ranking + time utilities)
- Handler typing via interfaces, not Zod
- Hook stale-counting stays synchronous but parses frontmatter for real timestamps
- CONSOLIDATION_AGE_LIMIT_DAYS extracted to shared constant

### Rejected features (all three reviewers agreed)
Checkpoint pruning, plan templates, export/reporting, tiered rollups, CLAUDE.md promotion tool, cross-project plan dependencies
