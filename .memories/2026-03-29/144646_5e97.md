## Merge-friendly memory format (v6.4.0)

Completed full implementation of merge-friendly memory format, replacing MEMORY.md with memory.yaml.

### Changes

- **memory.yaml replaces MEMORY.md**: Structured YAML with four fixed section keys (`decisions`, `open_questions`, `deferred_work`, `gotchas`), one entry per line (`"YYYY-MM-DD | description"`), chronologically sorted
- **Consolidation state moved to `~/.goldfish/`**: Machine-local at `~/.goldfish/consolidation-state/{name}_{hash}.json`, no more VCS merge conflicts on the cursor
- **Migration fallbacks everywhere**: `readMemory()` tries memory.yaml then MEMORY.md; `readConsolidationState()` tries new path then legacy; consolidation prompt tells subagent to check MEMORY.md as baseline if memory.yaml missing
- **All consumers updated**: recall, handlers, hooks, tool descriptions, skills, CLAUDE.md

### Post-review fixes (from Codex + Gemini review)

- Workspace collision: path hash suffix prevents two projects with same dir name from stomping each other's state
- Hook normalization: hooks import `getConsolidationStatePath` from src instead of re-implementing
- Parsing robustness: malformed JSON falls through to legacy, YAML detection handles leading whitespace, summary extracts entry text not raw YAML keys
- mkdir safety: handler ensures consolidation-state dir exists before subagent writes
- YAML recall tests added, stale doc references cleaned up
