## Prepared Goldfish 7.4.2 after source archive smoke failure

### WHAT
Discovered that the already-published `v7.4.1` GitHub source tarball failed `bun install --frozen-lockfile` because the package `prepare` script unconditionally ran `git config core.hooksPath .githooks` outside a git checkout. Fixed the script and bumped the release to `7.4.2` rather than rewriting the public `v7.4.1` tag.

### WHY
Release verification must include the downloaded published artifact. GitHub-generated source archives are not git worktrees, so an unconditional git hook setup in `prepare` breaks source-archive install smoke.

### HOW
- Added a regression in `tests/server.test.ts` that copies the package `prepare` script into a temporary non-git directory and runs `bun run prepare`.
- Red check failed with exit 128 against the old script.
- Changed `prepare` to `git rev-parse --is-inside-work-tree >/dev/null 2>&1 && git config core.hooksPath .githooks || true`.
- Bumped active release surfaces from `7.4.1` to `7.4.2` and added a `CHANGELOG.md` entry for source archive install safety.

### IMPACT
`v7.4.2` is the intended verified release. `v7.4.1` exists publicly but should be treated as superseded by `v7.4.2` because its generated source archive install smoke failed.
