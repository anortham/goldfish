## Fixed case-sensitive registry dedup creating duplicate Windows entries

A user reported recall misbehavior (perceived hang) with a registry.json containing mixed-case drive letters (`c:` vs `C:`) and mixed separators. Investigation confirmed goldfish creates these duplicates itself.

- **WHAT:** Registry path comparison is now case-insensitive for Windows paths in `registerProject`, `unregisterProject`, and `listRegisteredProjects` (which also collapses pre-existing case/separator-variant duplicates on read, first occurrence wins, file not rewritten).
- **WHY:** `registerProject` deduped with a case-sensitive string compare (separators normalized, case not). Every checkpoint save auto-registers using whatever drive-letter casing the harness's `process.cwd()` reports, so `c:/source/proj` and `C:/source/proj` accumulated as separate entries. Cross-workspace recall (`workspace: 'all'`, used by /standup) then scanned the project twice: duplicate checkpoints in output, duplicate workspace summaries, doubled full-corpus Orama indexing in search mode.
- **HOW:** New `registryPathKey()` delegates to the existing `normalizePathKeyForSafetyCheck` from workspace.ts — Windows paths get `win32.normalize` + lowercase, POSIX paths stay case-sensitive (correct: POSIX filesystems are case-sensitive). Reproduced first with a script registering `C:\source\demo-project` / `c:\source\demo-project` / `C:/source/demo-project` → 2 entries before, 1 after.
- **IMPACT:** Mixed-case registries self-heal on read; no manual registry.json editing needed. The user's backslash entries were a red herring (leftovers from pre-5.x storage, already handled). The one true unbounded wait found during investigation is the MCP `roots/list` request in server.ts hydration (no timeout, client-dependent) — separate issue, not addressed here.

Test note: 14-16 full-suite failures on this Windows machine are pre-existing flakes (lock/concurrency timing and workspace-resolution tests), identical on clean main; verified by name-diffing fail lists between baseline and fix.
