## Prepared 7.0.4 brief delete release

Goldfish now exposes the existing brief deletion capability through the MCP `brief` tool surface instead of leaving it as an internal API.

- **WHAT:** Added `delete` handling to `handleBrief()`, documented the `delete` action in `src/tools.ts`, and bumped release surfaces to `7.0.4`.
- **WHY:** The storage layer already supported deletion, but agents could not discover or call it through the published MCP schema.
- **HOW:** Added handler tests for confirmation, storage removal, active marker cleanup, error cases, and server tests that pin delete action documentation plus the explicit `id` requirement.
- **IMPACT:** Agents can now delete briefs through the supported MCP interface without relying on undocumented internals.
