## Fixed structured checkpoint search regression and prepared the 7.0.3 release

The latest ranking patch fixed narrative-description search on structured checkpoints, but it also dropped `briefId` and legacy `planId` terms from the indexed search surface.

- **Root cause:** `src/ranking.ts` switched the indexed `description` field from `buildRetrievalDigest()` to raw `checkpoint.description`, which removed digest-only metadata from search
- **Fix:** Added red regression tests for `briefId` and `planId` lookups, then restored that search surface with an explicit indexed `brief` field while keeping the raw description, async Orama calls, and deterministic tie-breaking
- **Release prep:** Bumped the enforced version surfaces to `7.0.3` and added a changelog entry for the search fixes
- **Verified:** `bun test --reporter=dot` -> 447 pass, `bunx tsc --noEmit` -> success

## Impact

Search keeps the structured-checkpoint fix without regressing brief-linked recall, and the repo is staged for a patch release push once the remaining staging decision is resolved.
