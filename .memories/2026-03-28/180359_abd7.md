## Delta checkpoints (v6.3.6)

- **What**: Added delta guidance to checkpoint tool description and cleaned up pre-compact hook message
- **Why**: Agents were writing cumulative "state of the world" checkpoints, wasting generation tokens on 90% repeated content
- **How**: Tool description now tells agents to capture only new progress since their last checkpoint. Pre-compact hook focuses on next steps and open questions instead of "include everything."
- **Decision**: Tool description is the right intervention point because the description is composed before the tool call. Server-side dedup would be too late.
- **Impact**: Should reduce token waste at checkpoint composition time and improve recall signal-to-noise ratio
