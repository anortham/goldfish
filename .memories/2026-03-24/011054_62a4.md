## Goldfish v6.1.0: Lean MEMORY.md consolidation + bug fixes

### Bug fix: consolidation batch cursor (v6.0.3)
The consolidation prompt told the subagent to write "now" as the `.last-consolidated` timestamp. When processing batches of 50 from a larger set, this jumped the cursor past all remaining checkpoints, silently skipping them. Fixed to use the last batch checkpoint's timestamp instead.

### Feature: `consolidate({ all: true })` (v6.0.3)
New parameter bypasses the 50-checkpoint batch cap for one-shot consolidation. Does NOT bypass the age window.

### Feature: 30-day age filter (v6.1.0)
Checkpoints older than 30 days are excluded from consolidation. They remain in `.memories/` for semantic search but won't be synthesized into MEMORY.md. The lifecycle: fresh checkpoints (days) for session continuity, consolidated into MEMORY.md (weeks) for decisions/rationale, then graduated to CLAUDE.md (months) or discoverable via search.

### Feature: Lean consolidation prompt (v6.1.0)
Rewrote the consolidation prompt with a strict content filter. Litmus test: "if you can derive it from the codebase, git log, or tools, it doesn't belong in MEMORY.md." Traffic light line budget: green (<25), yellow (25-40), red (>40). No prescribed section template. Expected to shrink typical MEMORY.md from 150+ lines to 15-25 lines.

- **Impact:** Significantly reduces token waste at session start. MEMORY.md now carries only decisions, rationale, open questions, deferred work, and gotchas.
- **Tests:** 420 pass, 0 fail
