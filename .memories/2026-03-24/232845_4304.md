## Fixed inflated stale checkpoint count in recall (v6.3.2)

The `recall()` function was counting all unconsolidated checkpoints regardless of age when reporting `staleCheckpoints`, while the `consolidate` handler properly filters to the last 30 days. This mismatch caused inflated counts (e.g. "216 checkpoints since last") triggering unnecessary consolidation runs.

- **Root cause:** `recall.ts` lines 557-570 had no age filter on the stale count, but `consolidate.ts` lines 54-57 applied `CONSOLIDATION_AGE_LIMIT_DAYS = 30`
- **Fix:** Added the same 30-day age filter to `recallFromWorkspace()` stale checkpoint counting
- **Tests:** Two new tests covering mixed old/recent checkpoints and all-old-checkpoints scenarios
- **Impact:** Recall now reports accurate stale counts matching what consolidate will actually process
