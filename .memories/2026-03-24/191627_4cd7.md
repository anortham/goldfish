## Deep code review and comprehensive hardening (v6.3.0)

Triggered by a Copilot user bug report where `planId` parameter was silently ignored (handler expected `id`).

### What we did
- **Review phase:** Spawned a 5-agent review team (storage, recall, plans, server, plugin), each doing a deep read-only review of their functional area
- **Fix phase:** Spawned a 4-agent fix team (storage, recall, plans, plugin), each owning distinct files to avoid conflicts
- **Logging:** Added file-based logging infrastructure (`src/logger.ts`) writing to `~/.goldfish/logs/`

### Key fixes
- **4 bugs:** Stale lock infinite loop, Math.min(...spread) RangeError on large datasets, savePlan TOCTOU race, plan ID path traversal
- **5 perf:** Recall short-circuits on limit:0, lazy stale counting, batched embeddings (8x), lightweight cross-workspace, compact consolidation JSON
- **9 quality:** Em dash removal, dead type cleanup, silent age limit reporting, error handling tightening, summary markdown stripping, git cwd parameter, memory timestamp fix
- **2 platform:** Registry mkdir asymmetry, Windows mixed path separators
- **5 plugin:** Skill docs updated for planId alias, recall consolidation pointer, includeMemory docs, stale .mcp.json removed, hook logic deduplicated

### Impact
- 476 tests (up from 427), 49 new tests added
- 42 files changed, +1495/-212 lines
- Agent teams workflow validated (TeamCreate, shared task lists, SendMessage coordination)
- File logging gives us diagnostic capability we never had
