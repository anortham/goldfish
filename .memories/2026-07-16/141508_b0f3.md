## Hooks tier design approved — implementation starting

**WHAT:** Design for reintroducing a hooks tier to Goldfish approved and committed as `docs/plans/2026-07-16-hooks-tier-design.md` (first commit on branch `worktree-hooks-tier`, worktree `.claude/worktrees/hooks-tier`).

**WHY:** Harness changes gutted the ambient behavioral surface: 2k cap on MCP server instructions + deferred tool loading hides tool descriptions until searched — sessions ignore goldfish tools.

**HOW (key decisions):**
- SessionStart only, matcher `startup|clear|compact` (`resume` excluded — resumed transcripts already contain the prior injection; re-firing duplicates)
- Static content only — no dynamic recall injection (user wants sessions that don't recall but still know to checkpoint)
- **Branchless hook script** — verified Codex accepts plain stdout as SessionStart context, same as Claude Code; ponytail's output adapter is unnecessary
- One shared `hooks/goldfish-hooks.json` referenced by `.claude-plugin/plugin.json` and new `.codex-plugin/plugin.json`; Codex sets `CLAUDE_PLUGIN_ROOT` as compat alias
- Codex manifest bundles `mcpServers` (`.codex-plugin/mcp.json`) — one install = tools + skills + hooks; trust-review step documented
- `getHookContext()` composed at runtime from `getInstructions()` verbatim + tool advertisement + quick reference + WHAT/WHY/HOW/IMPACT; ≤10k cap test-enforced
- `.codex-plugin/plugin.json` becomes 6th synced version surface

**IMPACT:** External Codex review (verdict: keep tier, fix gaps) materially improved the design — adapter deleted, mcpServers bundled, resume dropped. Gemini permanently unavailable (auth dead, user says Codex-only). Implementation dispatching now via subagent-driven-development lightweight path.
