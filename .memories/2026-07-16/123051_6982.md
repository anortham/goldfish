## Track 1: fixed cross-workspace recall cap + skill namespace bug + doc contradictions

First of four review-fix tracks (plan: docs/plans/2026-07-16-review-fixes-plan.md).

- **Cross-workspace recall bug:** `recall({workspace:'all'})` without date params silently capped every workspace at 5 checkpoints and under-reported `checkpointCount`. Root cause: `delete unlimitedOptions.limit` selected the default-5 path in `loadWorkspaceCheckpoints`. Fix: explicit `loadAll` parameter; caller states intent instead of simulating "no limit" by deleting the key. Regression test added (no-date-params, 7 checkpoints in one workspace, expects true counts) — watched it fail with 6/8 before the fix.
- **Skill tool-namespace bug:** all 6 skills declared `allowed-tools: mcp__goldfish__*` and used that prefix in ~15 examples, but plugin installs namespace tools as `mcp__plugin_goldfish_goldfish__*`. Dropped `allowed-tools` (ponytail-proven pattern) and shortened examples to bare tool names (`recall({...})`), matching server-instructions style. `.agents/skills` mirror resynced.
- **Checkpoint skill contradiction:** "Committed or pushed work — natural checkpoint moment after code lands" directly contradicted the checkpoint-before-commit rule; now says checkpoint BEFORE the commit so the file rides along.
- **Dead hook reference:** removed "(PreCompact hook handles this automatically)" — hooks were removed in v7.0.
- **Threshold docs:** CLAUDE.md and recall skill said "14+ days stale"; code is stale at >7d inactivity (STALE_BRIEF_DAYS) with a separate 14d refresh nudge (BRIEF_REFRESH_DAYS). Both docs now describe both thresholds.
