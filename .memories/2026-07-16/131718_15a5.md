## Codex review of the four review-fix commits: 6 findings, all verified and fixed

Codex (read-only, full 6df2540..21c3eea diff) returned no critical findings, 3 important + 3 minor, verdict "ready to merge with fixes". Each finding was verified against the code before implementing; all six were legitimate.

- **Fingerprint hardening (important):** day-cache fingerprint was name+size+mtime — a same-size edit with restored mtime could serve stale content. Added ctimeMs + ino (ctime can't be set by utimes; atomic rewrites change the inode). Regression test edits a file in place, restores mtime via utimes, and expects fresh content.
- **Full-mode budget hard cap (important):** the guaranteed first checkpoint could alone bust the 20k budget (single 60k description). It now truncates in place with an explicit note. Test covers one oversized checkpoint.
- **Stale-notice honesty (important, two parts):** (a) gist skipped nothing — briefs starting "## Goal" produced "Gist: Goal"; snippet now skips heading lines. (b) beyond the 28d scan window the notice claimed an exact age it never verified ("untouched 99d" with unverified 40d-old activity); it now reports the proven lower bound ("untouched 28d+") via StaleBriefNotice.scanWindowDays.
- **Containment dedup false positives (minor):** substring matching swallowed short tags inside unrelated words ("api" in "Capitalize"); dedup is now word-boundary-aware.
- **Version guard multi-tag (minor):** `git describe` picks one tag, so a non-release tag could shadow a mismatched release tag; checkVersionAgainstTags now inspects every tag from `git tag --points-at HEAD`.
- **Cached-object mutability (minor):** day cache now returns shallow copies of checkpoint objects (nested arrays documented read-only) so callers can't mutate cached entries.

Codex explicitly cleared the registry fast path, parallel git reads, and concurrent index builds of result-corruption issues.
