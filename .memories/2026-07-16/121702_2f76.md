## Project review: 4 focus areas audited, 1 confirmed data-loss bug

Full review of behavioral language, brief staleness, harness support (vs ponytail), and perf/token waste. Findings verified against source; perf numbers measured on synthetic corpora driving real code.

- **Confirmed bug:** `recall({workspace:'all'})` without date params silently caps every workspace at 5 checkpoints — `delete unlimitedOptions.limit` (recall.ts:628) selects the default 5 in loadWorkspaceCheckpoints (recall.ts:413). Comment says the opposite. All cross-workspace tests pass `days:1`, dodging the branch.
- **Checkpoint-before-commit:** instructions.ts:23 and tools.ts:22 push it correctly, but skills/checkpoint/SKILL.md:15 says checkpoint "after code lands" (contradiction), and :17 cites a PreCompact hook removed in v7.0. Checkpoint response never shows the saved file path or a commit-inclusion reminder.
- **Stale briefs root cause:** briefs only completed when replaced (v7.0.0 brief lived 2.5 months); checkpoint affinity counts as activity and masks staleness; nudges fire only at recall (session start), never at checkpoint-save or release moments. Code stale threshold is 7d (STALE_BRIEF_DAYS, recall.ts:27) but CLAUDE.md + recall skill document 14d.
- **Harness support:** all 6 skills declare `allowed-tools: mcp__goldfish__*` but plugin installs namespace tools `mcp__plugin_goldfish_goldfish__*` — allowed-tools never matches under the plugin install the skills ship with. Ponytail comparison: adopt its discipline (generators, drift guards, tag-equality version check), not its 22-harness breadth — its payload is prose, ours is a stateful MCP server. AGENTS.md is a contributor-guide copy of CLAUDE.md, wasting the zero-setup instruction tier.
- **Perf:** search re-reads + re-indexes corpus per call (247ms @1k vs 50ms target; prebuilt-index query is 0.8ms — evidence for an mtime-keyed cache); 5 serial spawnSync git calls eat ~40ms of the 50ms save budget and block the event loop; `recall({limit:0})` scans date dirs for brief activity (63x slower with stale brief); registry lock taken on every save.
- **Tokens:** `recall({limit:20, full:true})` ≈ 13k tokens, uncapped; compact search digest repeats decision/heading (~27% of 220-char budget); instructions/tool description duplicate the "checkpoint when" list; recall description at 1,926/2,000 chars.
