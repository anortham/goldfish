## Track 2: brief adoption nudges at the moments that matter

Second review-fix track (plan: docs/plans/2026-07-16-review-fixes-plan.md). Root cause being fixed: every brief nudge fired at recall (session start); nothing fired at checkpoint-save or completion moments, so briefs only got completed when replaced.

- **Commit reminder in checkpoint response:** `saveCheckpoint` now returns `filePath`; the handler renders `Saved: .memories/<date>/<file>.md — include this file in your git commit.` Reinforces checkpoint-before-commit exactly when the agent decides what to stage.
- **Brief-freshness nudge in checkpoint response:** when the active brief's text hasn't been updated in 7+ days (`BRIEF_UPDATE_NUDGE_DAYS`, half of recall's 14d refresh threshold), the save response asks "still the direction? Update or complete it." Catches drift at the engaged moment instead of the next session start.
- **Informed stale notice:** `StaleBriefNotice` gains a `snippet` (first non-heading content line, 120-char cap) so the recall stale nudge shows a gist — complete-vs-update is decidable without a follow-up `brief get`. Notice wording now offers update as a first-class option.
- **Lifecycle triggers in server instructions:** Briefs section was save-only; now: "update it when goals or constraints shift, complete it when the work lands, archive it when superseded." Instructions trimmed to 1,763/2,000 chars by removing the space-out sentence and /recall line duplicated in tool descriptions.
- **Documented deliberate duplication:** the checkpoint trigger list stays in both instructions and tool description because non-Claude MCP clients may never surface server instructions (noted in CLAUDE.md).
