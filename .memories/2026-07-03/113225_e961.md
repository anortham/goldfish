## Approved design for Goldfish 7.4.0: intent-blame recall + brief refresh nudge

Brainstormed the next version with the user and landed an approved design, committed to `docs/plans/2026-07-03-v7.4-intent-blame-design.md`.

- **WHAT:** Two features for 7.4.0. (1) Brief refresh nudge: catches zombie briefs — the 7.1.0 stale detector keys off checkpoint affinity, but checkpoints auto-inherit the active briefId, so a finished brief self-perpetuates forever (evidence: the v7.0.0 April brief still surfacing as active in July). New check on `brief.updated` age > 14 days emits a one-line reaffirm/complete nudge while keeping the body. (2) `file:` + `symbol:` recall filters: intent-blame queries over already-captured `git.files` and `symbols` frontmatter — case-insensitive path-suffix match with `/`-boundary guard, and exact symbol match. Both join hasStructuredFilter (full-corpus scan) and compose with existing filters.
- **WHY:** Both gaps are evidence-backed from this repo's own usage; "git for intent" positioning demands the ledger answer why-did-this-change queries.
- **HOW:** Follows the 7.2.0 type/tags filter and 7.1.0 staleness extension patterns. No new tools (3-tool constraint), non-destructive, no storage change. Skill guidance sweep to boost symbols: capture. Architecture risk: low.
- **IMPACT:** Next step is writing-plans → implementation plan, then TDD execution as 7.4.0.
