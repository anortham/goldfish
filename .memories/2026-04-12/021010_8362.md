## Fixed test pollution into ~/.goldfish and cleaned 31k junk entries

User noticed ~/.goldfish was full of junk. Root cause: tests leaking into the real home dir.

- **WHAT:** Added `GOLDFISH_HOME` temp-dir isolation to `tests/handlers.test.ts` and `tests/recall.test.ts` (beforeAll creates `mkdtemp`, afterAll restores env + removes dir). Purged `~/.goldfish/registry.json` from 5.6MB/31,536 entries to 4KB/12 real projects. Deleted 655 `test-*` and `goldfish-consolidate-*` files from `~/.goldfish/consolidation-state/` (658 → 3). Removed 0-byte legacy `~/.goldfish/index.db`.
- **WHY:** `saveCheckpoint()` calls `registerProject()` fire-and-forget at `checkpoints.ts:448`. recall and handlers tests never set `GOLDFISH_HOME`, so registry + consolidation-state writes leaked to the real home dir. Years of `/tmp/test-*` and `/var/folders/**/test-handlers-*` entries bloated registry.json.
- **HOW:** Filtered registry via bun one-liner matching `/(tmp|private\/var\/folders|var\/folders)/` paths and `^(test-|goldfish-consolidate-)` names. Used `find -delete` for consolidation-state cleanup.
- **IMPACT:** All 578 tests pass. Post-test registry.json stays at 4KB (was growing unbounded per test run). Future test runs no longer pollute the user's real Goldfish state. About to commit.
