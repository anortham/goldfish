## Updated OpenCode to plugin-based Superpowers install

- **WHAT:** Migrated the local OpenCode setup from the old symlink-based superpowers install to the plugin-based install described in the upstream `.opencode/INSTALL.md`.
- **WHY:** The upstream superpowers repo changed its OpenCode installation flow, so the local setup needed to follow the supported plugin path instead of continuing to rely on manual symlinks.
- **HOW:** Added `superpowers@git+https://github.com/obra/superpowers.git` to `~/.config/opencode/opencode.json`, removed the legacy symlink entries from `~/.config/opencode/plugins/` and `~/.config/opencode/skills/`, then started fresh OpenCode runs to force package installation and confirm skill discovery.
- **IMPACT:** Future OpenCode runs now load superpowers from the plugin manager and discover skills from the cached plugin package. The old cloned repo is still present as an inert leftover, but it is no longer part of the active installation path.
