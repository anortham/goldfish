## WHAT

Prepared the `7.0.1` patch release and cleaned up the post-v7 surface.

- Fixed the typecheck regressions that were hiding behind a green Bun test run
- Removed stale v7 wording about fuzzy search, consolidation, and `/consolidate`
- Documented the Codex Desktop setup caveat in `README.md`, including the project-local `.codex/config.toml` plus `GOLDFISH_WORKSPACE` requirement
- Bumped the release version across all enforced surfaces and added a changelog entry

## WHY

The repo was in an awkward state where runtime tests passed but `bun run typecheck` failed, and the agent-facing docs still described deleted behavior. That makes the release surface lie about what shipped and invites broken client setup.

## HOW

- Tightened server/README regression tests first so the failures were explicit
- Fixed `exactOptionalPropertyTypes` issues in workspace hydration, recall option normalization, and git spawn options
- Updated canonical and mirrored recall skills together so repo-local skill discovery stays in sync
- Bumped `SERVER_VERSION`, `package.json`, plugin metadata, README banner, and `CHANGELOG.md` to `7.0.1`

## IMPACT

- `bun test` and `bun x tsc --noEmit` are both green again
- Goldfish's public docs and skill text now match the real v7 feature set
- Codex Desktop users now get a concrete workspace-binding setup instead of silent misconfiguration
