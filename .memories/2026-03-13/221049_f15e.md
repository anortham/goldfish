## Goldfish v5.9.0 — Semantic Search Efficiency Overhaul

Dogfooded the semantic search system, found 3 issues, designed and implemented fixes across 11 commits.

### What Changed

**1. Uncapped semantic maintenance** — removed the 3-item/150ms budget cap from `runSearchSemanticMaintenance`. First `recall({ search })` now indexes the entire pending backlog in one pass. Made `maxMs` optional in `processPendingSemanticWork` to avoid `setTimeout(fn, Infinity)` overflow (32-bit int clamp fires immediately).

**2. Relevance floor** — refactored `buildHybridRanking` to return `ScoredCheckpoint[]` with computed scores. Added `MINIMUM_SEARCH_RELEVANCE = 0.15` threshold. Off-topic queries now return empty results instead of low-quality filler. Also fixed an edge case where empty lexical orders gave max scores to all candidates.

**3. Cache cleanup on startup** — added `workspacePath` field to semantic manifests (with `readManifest` fix to preserve it on round-trip). `pruneOrphanedSemanticCaches()` runs fire-and-forget on server start, deleting dirs with no manifest, no workspacePath, or nonexistent paths. Caps at 500 dirs/startup.

**4. Test isolation fix** — added `GOLDFISH_HOME` env var support to `getGoldfishHomeDir()`. Fixed cross-file test interference where `workspace.test.ts` mutating `HOME` caused `semantic-cache.test.ts` concurrent upsert test to fail in full suite.
