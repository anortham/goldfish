## Token waste audit and fixes — v5.8.3

Audited Goldfish for token inefficiencies in MCP output. Identified and fixed two high-impact issues:

### 1. Duplicate MCP instructions (~500 tokens/session)
- Both `.mcp.json` and `.claude-plugin/plugin.json` registered the same server
- Claude Code injected `getInstructions()` output twice in system prompt
- **Fix:** Deleted `.mcp.json` — plugin registration is the canonical path

### 2. Structured field redundancy in compact recall (~200 tokens/checkpoint)
- `presentCheckpoint` in non-full mode only stripped `git` info
- All forensic fields (context, decision, alternatives, evidence, impact, symbols, unknowns, confidence) still appeared alongside the compact description
- **Fix:** Strip forensic fields in non-full mode, keep only orientation fields (tags, type, planId, next)

**Impact:** ~1500-2000 fewer tokens per session. Compact recall output went from ~13 metadata lines per checkpoint down to ~3-4.
