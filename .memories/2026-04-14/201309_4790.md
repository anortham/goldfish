## Recall input hardening shipped as v6.5.3

Hardened `recall()` against placeholder inputs from tool-calling agents (an agent was tripping over `days: 0` and empty-string date fields).

- **What:** Added `normalizeRecallOptions` in `src/recall.ts` — trims blank strings (`workspace`, `since`, `from`, `to`, `search`, `planId`) to undefined and drops non-finite/zero/negative `days`. Ran once at entry to `recall()` so MCP handler and direct TS callers share the behavior.
- **Why:** Agents send `0` and `""` as placeholders instead of omitting fields. `days: 0` created a zero-width date window; empty `since/from/to` pushed recall into date mode with `Invalid Date` errors.
- **How:** Single normalization point per the approved design doc (`docs/plans/2026-04-14-recall-input-hardening-design.md`). `limit: 0` semantics preserved (still plan-only). `since`-beats-`days` precedence preserved.
- **Tests:** New regression coverage for `days: 0`, negative days, blank date strings, blank search, and `since` precedence. Replaced the old `days: 0 = explicit filter` test.
- **Version bump:** 6.5.2 → 6.5.3 across `package.json`, `.claude-plugin/plugin.json`, `.claude-plugin/marketplace.json`, `src/server.ts`, `README.md`.
- **Impact:** 582 tests pass. API is forgiving of sloppy agent inputs without changing semantics for valid callers.
