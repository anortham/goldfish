## Diagnosed SessionStart hook errors on Windows

Both goldfish and superpowers plugins show "SessionStart:startup hook error" on session start.

- **Root cause:** Known Claude Code bugs (#19491, #12671) where plugin SessionStart hooks run before plugins are fully loaded
- **Windows compounds it:** Additional issues #9542, #23038 with SessionStart hooks on Windows
- **Impact:** Cosmetic only — both plugins work fine despite the errors. MCP servers and skills load through different mechanisms.
- **Workaround:** Could move goldfish's SessionStart hook to user-level `~/.claude/hooks/hooks.json`
- **Proper fix:** Needs to come from Claude Code team
