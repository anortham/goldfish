# Semantic Search Context Exploration - Complete Report

## Executive Summary

Explored the Goldfish project to understand:
1. Current search implementation using fuse.js (keyword-based fuzzy matching)
2. Historical semantic search implementations that were stripped out
3. RAG (Retrieval-Augmented Generation) design approach from Nov 2025
4. Current architecture decisions favoring simplicity

## Key Findings

### 1. Current Search Implementation (ACTIVE)

**Location:** `src/recall.ts` lines 56-78

**Current approach:**
- Uses fuse.js for fuzzy string matching across checkpoint descriptions
- Searches across: description (weight: 2), tags (weight: 1), git.branch (0.5), git.files (0.3)
- Threshold: 0.4 (0=perfect match, 1=match anything)
- Ignores location in text (searches anywhere)
- Returns results sorted by fuse.js relevance score

**Limitations:**
- Keyword-only matching (cannot understand semantic meaning)
- Cannot find related concepts (e.g., "JWT bug" won't match "authentication issue")
- Returns summaries by default to save tokens (lossy)

### 2. Historical Semantic Search Implementation

**Stripped in commit 3e55d8f (Feb 14, 2026)**
- Removed 6,522 lines of complex infrastructure
- Removed dependencies: hnswlib-node, @xenova/transformers, onnxruntime-node, sqlite-vec
- Removed platform binaries for linux, macos-arm64, macos-intel, windows
- Reduced runtime dependencies from 8 to 3: MCP SDK, fuse.js, yaml

**What was stripped:**
- `src/embeddings.ts` (473 lines) - Embedding generation with ONNX models
- `src/database/embeddings.ts` (473 lines) - Vector storage in SQLite
- `src/distill.ts` (283 lines) - LLM distillation via Claude/Gemini CLI
- `src/storage/` (JSONL checkpoint storage with hashing)
- `src/sync/` (Cross-workspace sync engine)
- Migration scripts (migrate-embeddings, migrate-to-memories)
- 188 tests for embeddings, distillation, semantic search

**Why it was stripped:**
- Added complexity with native binaries and model downloads
- Distillation required external CLI tools (claude/gemini commands)
- Philosophy shift: keep Goldfish dumb, let Claude (the LLM) be smart
- Result: 265 tests pass with zero native dependencies

### 3. RAG Design (Nov 2025 - docs/RAG_PLANNING.md)

**Two-stage pipeline (since removed but documented):**

**Stage 1: Semantic Retrieval**
- Embed checkpoints using BGE-Small-EN-V1.5 (384 dimensions)
- Store vectors in SQLite with HNSW index (fast nearest-neighbor search)
- On query: embed user query, vector search Top-K results
- Find conceptually similar work, not just keywords

**Stage 2: LLM Distillation**
- Take Top-K retrieved checkpoints (potentially 4,000+ tokens)
- Call Claude or Gemini CLI with special distillation prompt
- LLM synthesizes into concise, query-specific summary (150-500 tokens)
- Cache results to avoid redundant LLM calls
- Fallback chain: Claude → Gemini → simple extraction

**Performance targets (from RAG plan):**
- Embedding: <50ms/checkpoint
- Vector search: <100ms for 1000 checkpoints
- Claude distillation: <1.5s
- Total recall with distillation: <2.5s

### 4. Current Summary Generation (ACTIVE)

**Location:** `src/summary.ts`

Simple first-sentence extraction:
- Threshold: 150 chars (no summary for short descriptions)
- Extract first sentence (split on `.` or newline)
- Truncate at 147 chars + "..." if needed
- **Result:** 70-150 character summaries in checkpoint YAML frontmatter

Used in `recall.ts` to reduce context when `full: false` (the default)

## Architecture Philosophy (From docs/plans/2026-02-14-revival-design.md)

Key decision: **Strip embeddings and distillation, keep Goldfish dumb**

Rationale:
1. **Dumb storage + Smart skills** - MCP server stores/retrieves, skills contain behavior
2. **Extension point for future** - RAG design documented, can be revived when needed
3. **Zero native dependencies** - No model downloads, platform binaries, or native modules
4. **Let Claude be smart** - Agents know how to filter and distill context themselves
5. **Radical simplicity** - ~2,000 LOC, 265 tests, fast, maintainable
