## Brainstormed semantic search for Goldfish — decided to park it

Explored adding semantic search (Wasm embeddings via @huggingface/transformers) to make project reasoning/decisions queryable alongside code understanding from Julie.

### What was discussed
- **Goal:** Let agents ask "why was this decision made?" and find relevant memories + project docs
- **Approach explored:** Wasm embeddings (all-MiniLM-L6-v2), hybrid indexing (eager for checkpoints, lazy for docs), zero native deps
- **Corpus:** All non-code textual files (memories, plans, design docs, README, CLAUDE.md, etc.)

### Why we parked it
- **No evidence of need yet** — CLAUDE.md already covers key decisions, recall() surfaces recent context, Julie handles code
- **Complexity budget is real** — we stripped 6,500 lines of exactly this infrastructure 2 weeks ago
- **Project principle:** "only add complexity when we have EVIDENCE we need it"
- **Revisit trigger:** When there are concrete retrieval failures — specific moments where an agent needed context it couldn't find

### Key insight
The gap between "fuzzy keyword search" and "semantic search" matters most on large, long-lived projects with hundreds of decisions buried in old checkpoints. Goldfish and most projects using it aren't there yet.
